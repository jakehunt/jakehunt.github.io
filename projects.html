This Website

Description:

I created this website to learn about something I've always taken for granted: the internet. Along the way I ran into countless problems which tested my perseverance to continue. I didn't back down and became a better googler because of it. Whenever I thought of some functionality my website should have, I knew that it could be done.

Website Details:

This website is hosted on Github Pages and the domain name was acquired on GoDaddy.com. The canvas on the home page is created using processing.js which I learned on KhanAcademy. Some of this I learned on w3schools including Bootstrap v.4 which is used for the website's mobile functionality and responsiveness. Everything else was found from one-off google searches with many answers being found on stackOverflow.

The fourth iteration of this website includes heavy use of the load() function using ajax in jquery. This MAJOR improvement allows for repetitive code (like the navbar) to not be hard coded on every webpage. Rather, each webpage can "load" the navbar html code from a single page: in my case the index page. Whenever I make changes to the navbar, now I only have to make them in one place. This change was inspired by learning about the power of xrefs in AutoCAD.

Another major improvement made in version 4 was automating the html creation of the 'list' pages including the cool links, books, and projects. The insertAdjacentHTML() and jquery function .html() were both super-handy tools to getting this to work. Now, whenever I want to add to one of these lists, I just have to create a new object constructor and add its variable name to an array. Easy! No need to touch the html whatsoever.

Also, in version 3, each project file had its own webpage. This was unwieldy and discouraged me from adding new projects. I stumbled across deanebarker.net and realized on his reading list page, each review wasn't on its own webpage. That was the moment I realized I wasn't fully using the power of the CSS display rule. Now, when you select "See Review" on the book list webpage, or "See Project" on the project list webpage, the contents of the current webpage are hidden and the desired contents are loaded/shown! Brilliant!

"CharlieClock"

Description:


This project improves on the concept of multiplexing. To understand multiplexing, you must first understand 7-segment displays. In this project, I used a 4-digit, 7-segment LED display. Spoiler alert: only one segment of a digit is being displayed at a time!

If the arduino was instructed to light up all 4 digits at a time, they would appear very dim because it's running on a 5 volt circuit (a low voltage), and has low current output. The first improvement on this problem is to light up only 1 digit at a time, thus making each digit appear brighter while cycling through at a frequency faster than human perception. This is multiplexing.

However, in multiplexing, the arduino in instructed to light up an entire digit at a time. This causes displayed digits which use more segments (like '8') to appear more dim than digits with less segments (like '1'). This is due to the same amount of electrical power/current being utilized for the entire digit. To fix this, one must not only cycle the digits, but also the segments of the digits. This is charlieplexing. In this implementation, a segment lights up once every 300 microseconds which correlates to a display frequency of 3333.33 Hz.

To better see the reality of how the segments are being displayed, I added a button that changes the delay to 50 milliseconds which can be perceived by the human eye. See the video at bottom of page for reference. You'll notice that the single segment appears brighter. The problem with both multiplexing and charlieplexing is that as the delay time is decreased in the for loop between lighting up segments, the segments and digits appear dimmer. The only fix that I know of for this is to run the clock on a circuit which can deliver a higher current. If any reader sees this and knows another fix, send me an email!

Lastly, the system is able run autonomously plugged into the wall away from the computer because of the DS3231 clock module.
